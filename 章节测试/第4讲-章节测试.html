第4讲 决策树入门 — 章节测试

测试范围：决策树概念、熵与信息增益、ID3算法、C4.5算法、剪枝

一、选择题 (×7)

1决策树的学习过程通常被描述为一个自顶向下的递归过程。这种构建方式本质上属于哪种算法策略？
  1. 动态规划 (Dynamic Programming)
  2. 贪心算法 (Greedy Algorithm)
  3. 穷举搜索 (Exhaustive Search)
  4. 随机优化 (Stochastic Optimization)
答案：B — ID3/C4.5 每一步选择当前最优属性（信息增益最大/增益率最大），不回溯，属于贪心策略。
2在 ID3 算法中，使用信息增益作为分裂标准会导致算法倾向于选择以下哪类特征？
  1. 缺失值较少的特征
  2. 取值较少的特征（如：性别）
  3. 取值较多且分布稀疏的特征（如：身份证号、日期）
  4. 数值型特征
答案：C — 信息增益偏向取值多的特征。例如"编号"每个样本唯一，条件熵=0，信息增益=H(D)最大，但毫无泛化能力。
3为了修正信息增益的缺点，C4.5 算法引入了"信息增益率"。关于增益率，下列说法正确的是：
  1. 增益率完全解决了对取值较多特征的偏好
  2. 增益率在分母上增加了一个反映特征固有属性的熵（Intrinsic Value），从而惩罚了取值过多的特征
  3. 增益率倾向于选择取值极少的特征
  4. 增益率不再需要计算信息增益
答案：B — 增益率 = g(D,A) / HA(D)，分母 HA(D) 为特征 A 的固有熵。特征取值越多 → HA(D) 越大 → 增益率降低，从而起到惩罚作用。
4决策树生成的决策边界 (Decision Boundary) 具有什么物理几何特征？
  1. 决策边界总是平滑的曲线
  2. 决策边界是倾斜的超平面
  3. 决策边界总是与坐标轴平行或垂直的
  4. 决策边界呈现圆形或椭圆形
答案：C — 决策树每次分裂仅在单一特征上做条件判断（如 年龄 ≥ 30?），因此决策边界是与坐标轴平行或垂直的阶梯状界线。
5决策树在处理连续型特征（如年龄、收入）时，通常采取什么手段？
  1. 无法处理，必须先手动离散化
  2. 采用二分法 (Bi-partition) 寻找最优划分点
  3. 直接计算每个数值的信息增益
  4. 使用均值代替
答案：B — C4.5 使用二分法：将连续值排序，取相邻值的中间点作为候选划分点，选择使信息增益最大的划分点。
6预剪枝 (Pre-pruning) 和后剪枝 (Post-pruning) 相比，以下说法错误的是：
  1. 预剪枝在构建过程中提前停止，计算效率更高
  2. 预剪枝通常比后剪枝产生的树更大、更复杂
  3. 预剪枝可能导致"欠拟合"，因为它可能会错过后续更好的分裂
  4. 后剪枝通常保留了更多的细节，泛化性能往往优于预剪枝
答案：B — 预剪枝提前停止，产生的树更小、更简单（而非更大更复杂）。后剪枝先生成完整树再剪，泛化性能更好但计算开销更大。
7在训练决策树时，如果某些样本的某个特征缺失，C4.5 算法的处理方式是：
  1. 直接丢弃这些样本
  2. 用该特征的众数填充后再计算
  3. 根据已知该特征权重的比例，将样本分配到所有子节点中
  4. 随机选择一个子节点分配
答案：C — C4.5 按概率将缺失样本分配到各个子节点，而非简单丢弃或填充。这是 ID3 不具备的能力。

二、填空题 (×4)

1决策树本质上是if-then 规则的集合，从根节点到叶节点的每条路径对应一条规则，且任意两条路径之间互斥且完备。
答案： if-then 规则；互斥且完备
2ID3 算法以信息增益作为属性选择准则，C4.5 以信息增益率作为准则。
答案： 信息增益；信息增益率
3决策树是判别模型（生成/判别），它学习的是条件概率分布 P(Y|X)，而非联合分布 P(X,Y)。
答案： 判别
4剪枝的损失函数为 Cα(T) = C(T) + α|T|，其中 C(T) 衡量模型拟合程度，|T| 衡量模型复杂度，α 取值越大则决策树越小。
答案： 模型拟合程度；模型复杂度；小

三、简答题 (×3)

1简述 ID3 算法的三种停止条件，并说明每种情况下叶节点标签如何确定。
答案：
(1) 当前节点包含的样本全属于同一类别 → 将该节点标记为叶节点，类别为该类。
(2) 当前属性集为空（所有属性已用完）→ 标记为叶节点，类别为该节点样本的多数类。
(3) 当前节点包含的样本集合为空 → 标记为叶节点，类别为父节点样本的多数类。
2为什么 ID3 算法会产生偏向取值较多特征的问题？C4.5 是如何修正这一缺陷的？
答案：
ID3 使用信息增益 g(D,A) = H(D) − H(D|A) 选择特征。当特征取值越多时，每个取值对应的子集越"纯净"（熵越小），条件熵 H(D|A) 越接近 0，信息增益越接近 H(D) 最大。极端情况如"编号"：每个样本编号唯一，H(D|编号)=0，信息增益=H(D) 最大，但无泛化能力。

C4.5 使用信息增益率 gR(D,A) = g(D,A) / HA(D)，分母 HA(D) 是特征 A 的固有熵。特征取值越多 → HA(D) 越大 → 增益率被惩罚，从而修正偏好。
3简述决策树预剪枝和后剪枝的区别及各自优缺点。
答案：
预剪枝：在构建过程中提前停止分裂（如限制最大深度、叶节点最小样本数）。优点：计算效率高。缺点：可能欠拟合，错过后续更好的分裂。

后剪枝：先生成完整树，然后自下而上剪去不具代表性的子树，通过最小化损失函数 Cα(T)=C(T)+α|T| 来选择最优子树。优点：泛化性能更优。缺点：计算开销较大。

四、计算题 (×1)

1 给定训练数据集（买计算机），总样本 1024 个，其中买=641，不买=383。年龄属性的取值分布如下：
  青年(384样本)：买=128，不买=256
  中年(256样本)：买=256，不买=0
  老年(384样本)：买=257，不买=127
(1) 求数据集 D 的经验熵 H(D)。（保留4位小数）
(2) 求年龄属性对数据集 D 的条件熵 H(D|年龄)。
(3) 求年龄属性的信息增益 g(D, 年龄)。
(4) 青年子集中，若仅有"学生"和"信誉"两个候选属性，如何选择下一步分裂属性？（需计算说明）
答案：
(1) H(D) =
p(买) = 641/1024 = 0.6260, p(不买) = 383/1024 = 0.3740
H(D) = −0.6260×log2(0.6260) − 0.3740×log2(0.3740) = −0.6260×(−0.6764) − 0.3740×(−1.4190) = 0.4236 + 0.5301 = 0.9537

(2) H(D|年龄) =
青年: p(买)=128/384=1/3, H = −(1/3)log2(1/3) − (2/3)log2(2/3) = 0.9183
中年: p(买)=256/256=1, H = 0
老年: p(买)≈257/384≈0.6693, p(不买)≈127/384≈0.3307, H ≈ 0.9157
H(D|年龄) = (384/1024)×0.9183 + (256/1024)×0 + (384/1024)×0.9157
 = 0.375×0.9183 + 0.25×0 + 0.375×0.9157
 = 0.3444 + 0 + 0.3434 = 0.6877

(3) g(D, 年龄) = H(D) − H(D|年龄) = 0.9537 − 0.6877 = 0.2660

(4) 青年子集(384样本)：买=128, 不买=256，H(D青年)=0.9183
 • g(D, 学生) = 0.9183 − 0 = 0.9183 (学生属性完美划分)
 • g(D, 信誉) = 0.9183 − 0.8741 = 0.0441
选择学生作为下一个分裂属性（信息增益最大）。

五、代码分析题 (×1)

1阅读以下R语言代码，回答问题： 
library(rpart)
library(rpart.plot)

# 训练决策树
model <- rpart(Species ~ ., data = iris,
               method = "class",
               control = rpart.control(minsplit = 20, 
                                       cp = 0.01, 
                                       maxdepth = 5))

# 查看CP表
printcp(model)

# 预测
pred <- predict(model, iris, type = "class")
conf <- table(iris$Species, pred)
accuracy <- sum(diag(conf)) / sum(conf)
print(paste("准确率:", round(accuracy * 100, 2), "%"))

# 可视化
rpart.plot(model, type = 4, extra = 101)

(1) method = "class" 的含义是什么？如果不写这个参数，rpart会怎样？（4分）

(2) cp = 0.01 和 minsplit = 20 分别控制什么？增大 cp 值对树的大小有何影响？（4分）

(3) printcp(model) 输出的CP表中，如何根据CP值选择最优树？（4分）

答案：
(1) method = "class" 表示构建分类树（使用Gini指数作为不纯性度量）。如果不写，rpart会尝试推断：当目标变量为因子型时自动使用"class"，数值型时使用"anova"（回归树）。但显式指定更安全。

(2) cp（复杂度参数）控制剪枝程度——分裂必须使R²提升至少cp值。cp越大，允许的分裂越少，树越小。minsplit 表示节点继续分裂所需的最小样本数（默认20）。增大cp值会使决策树更小、更简单，是预剪枝的方式。

(3) CP表中关键列：CP（复杂度参数值）、nsplit（分裂次数）、rel error（相对误差）、xerror（交叉验证误差）、xstd（xerror的标准差）。选择xerror最小的CP值对应的树；或采用"1-SE规则"——选择xerror不超过 min(xerror)+xstd 的最简单（cp最大）的树。
