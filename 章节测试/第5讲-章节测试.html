第5讲 章节测试

决策树：CART算法、剪枝与集成学习入门 | 共 5 类题型

一、单选题（5题）

1 CART 决策树与 ID3/C4.5 在树结构上最重要的区别是什么？
A. CART 只能用于分类
B. CART 总是生成二叉树
C. CART 使用熵作为分裂准则
D. CART 不支持剪枝

正确答案：B

CART 采用二叉划分，而 ID3/C4.5 采用多叉划分。CART 不仅支持分类也支持回归任务，使用 Gini 指数作为分类分裂准则，并且需要剪枝。

2 在二分类问题中，当样本比例为 1:1 时，基尼系数的值是多少？
A. 0
B. 0.25
C. 0.5
D. 1

正确答案：C

Gini = 2p(1-p) = 2×0.5×0.5 = 0.5。当 p=0.5 时基尼指数最大，表示不确定性最高。

3 在构建回归树时，CART 使用什么指标来衡量分裂质量？
A. 交叉熵 Cross Entropy
B. 平方误差 Squared Error / 最小方差
C. 平均绝对误差 MAE
D. 相关系数 Correlation Coefficient

正确答案：B

CART 回归树采用最小平方残差（最小化平方误差和）作为分裂准则，寻找最优切分变量 j 和切分点 s 使得两个子区域内的平方误差之和最小。

4 随机森林在 Bagging 基础上增加的另一层随机性是什么？
A. 随机选择不同的损失函数
B. 随机选择不同的训练迭代次数
C. 在节点分裂时随机选择一个特征子集进行最优属性搜索
D. 随机对输入样本进行降维

正确答案：C

随机森林在 Bagging 的 Bootstrap 采样基础上，在每个节点分裂时从 M 个特征中随机选取 m ≪ M 个特征，进一步降低树间的相关性，从而进一步降低方差。

5 随机森林中，超参数 m（mtry，分裂时随机抽取的特征数）设置得非常大，接近总特征数 D 时，模型会怎样？
A. 模型变得更加随机
B. 随机森林退化为传统的 Bagging 算法
C. 模型计算速度大幅提升
D. 模型倾向于欠拟合

正确答案：B

当 m 接近全部特征数 D 时，每个节点几乎使用全部特征来分裂，丧失了特征层面的随机性，RF 退化为普通的决策树 Bagging。

6 相比于单棵决策树，随机森林最主要的缺点是什么？
A. 容易过拟合
B. 无法处理分类变量
C. 模型丧失了直观的可解释性
D. 无法处理高维数据

正确答案：C

单棵决策树可以直观画出树结构来理解分类/回归规则，而随机森林由大量树组成，是黑箱模型，失去了可解释性。

7 CART 剪枝的损失函数中，α 参数增大时，决策树会如何变化？
A. 决策树变大
B. 决策树变小
C. 决策树深度不变化
D. 决策树预测精度必然上升

正确答案：B

Cα(T) = C(T) + α|T|，α 越大，对复杂度的惩罚越大，倾向于选择叶结点更少的小树。α = 0 对应完全树 T₀，α → ∞ 对应单结点树。

二、填空题（4题）

1 CART 分类树使用 ____ 作为选择最优特征的不纯性度量，而 ID3 使用 ____。

答案：基尼指数（Gini Index）、信息增益（Information Gain）

2 Bootstrap 自助法中，当样本量 m 足够大时，每次采样不被抽到的概率约等于 ____，即每次约有 ____% 的样本进入训练集。

答案：1/e (≈ 0.368)、63.2

P(未被抽到) = (1-1/m)^m → 1/e ≈ 0.368，所以约 63.2% 进入训练集，36.8% 作为 袋外 (OOB)。

3 在构建随机森林时，基学习器（决策树）通常建议____（填"进行剪枝"或"完全生长不剪枝"）。

答案：完全生长不剪枝

随机森林每棵决策树都完全生长，不需要剪枝，因为随机性（样本随机+特征随机）已经起到了防止过拟合的作用。

4 CART 剪枝的损失函数为 Cα(T) = C(T) + α|T|，其中 C(T) 表示____，|T| 表示____。

答案：对训练数据的预测误差（基尼指数或均方误差）、子树 T 的叶结点总个数

三、简答题（3题）

1 简述 Bagging 为什么能够降低模型方差？其方差公式为 Var = ρσ² + (1-ρ)σ²/B，请解释各符号含义，并指出为什么增大 B 不能完全消除方差。

参考答案：

σ²：每个基学习器的方差
ρ：基学习器两两之间的相关系数
B：Bagging 中基学习器的个数

当 B 增大，第二项 (1-ρ)σ²/B → 0，但第一项 ρσ² 保持不变。因此增大 B 无法消除由基学习器之间相关性带来的方差部分。要进一步降低方差，需要降低 ρ（如随机森林通过随机化特征来降低树间相关性）。

2 简述 CART 剪枝中子树序列 {T₀, T₁, ..., Tₙ} 是如何生成的，最后如何从中选出最优子树。

参考答案：

(1) 子树序列生成：从完全生长树 T₀ 开始，对 T₀ 中每个内部节点 t，计算剪枝临界值 g(t) = [C(t)-C(T_t)]/[|T_t|-1]。每次剪掉 g(t) 最小的节点对应的子树，得到 T₁。以 T₁ 作为当前树继续迭代，直到只剩根节点，得到序列 T₀, T₁, ..., Tₙ。

(2) 选择最优子树：在独立的验证集上，对序列中每棵子树进行交叉验证，选择验证误差最小的子树作为最终决策树。

3 为什么随机森林通常比 Bagging 表现更好？请从方差降低的角度加以解释。

参考答案：

Bagging 的方差 = ρσ² + (1-ρ)σ²/B。虽然增大 B 可降低第二项，但第一项 ρσ² 由于 Bootstrap 样本之间的重叠而无法消除。

随机森林在每个节点分裂时，从 M 个特征中随机选取 m ≪ M 个特征再进行最优属性选择。这种特征层面的随机化使不同决策树之间的相关性 ρ 显著降低，从而降低集成方差中的 ρσ² 项，进一步提升性能。

四、计算题（1题）— 基尼指数计算

1 设数据集 D 共有 20 个样本，其中正类 12 个，负类 8 个。现考虑特征"是否已婚"（取值：是/否），已婚样本 10 个（其中正类 8 个，负类 2 个），未婚样本 10 个（其中正类 4 个，负类 6 个）。

(a) 计算数据集 D 的基尼指数 Gini(D)。
(b) 计算按"是否已婚"划分后的条件基尼指数 Gini(D, 是否已婚)。
(c) 该划分的基尼指数是否低于划分前？该特征是否是好的划分特征？

D = {正类:12, 负类:8}, 共20 已婚: 10个, {正类:8, 负类:2} → H1 未婚: 10个, {正类:4, 负类:6} → H2

参考答案：

(a) Gini(D) = 1 - [(12/20)² + (8/20)²] = 1 - [0.36 + 0.16] = 1 - 0.52 = 0.48 (b) Gini(已婚) = 1 - [(8/10)² + (2/10)²] = 1 - [0.64 + 0.04] = 0.32 Gini(未婚) = 1 - [(4/10)² + (6/10)²] = 1 - [0.16 + 0.36] = 0.48 Gini(D, 是否已婚) = (10/20)×0.32 + (10/20)×0.48 = 0.5×0.32 + 0.5×0.48 = 0.16 + 0.24 = 0.40 (c) 划分后的基尼指数 0.40 < 划分前的 0.48，基尼指数下降了。 这使得 Gini gain = 0.48 - 0.40 = 0.08 > 0，说明该特征有正向的划分效果。

五、代码分析题（1题，12分）

1 阅读以下R语言代码，回答问题：
library(rpart) # 特征矩阵与标签 set.seed(42) X <- matrix(c(2.0, 1.5, 3.0, 2.5, 1.0, 1.0, 2.0, 2.5, 1.0, 2.5), ncol = 2) y <- factor(c("否", "否", "是", "是", "否")) colnames(X) <- c("特征1", "特征2") data <- data.frame(X, y) # 训练CART分类树 model <- rpart(y ~ ., data = data, method = "class", control = rpart.control(minsplit = 3, cp = 0.01)) print(model) summary(model)

(1) CART分类树使用哪种不纯性度量？上述代码中method = "class"时rpart默认使用的是什么标准？（4分）

(2) rpart.control(minsplit = 3, cp = 0.01) 中各参数的含义是什么？如果数据集有1000个样本但minsplit = 3太小，会导致什么问题？（4分）

(3) summary(model) 输出的内容中，如何识别该决策树使用了哪些变量作为分裂节点？（4分）

参考答案：

(1) CART分类树使用Gini指数（Gini Index）作为不纯性度量。在 R 的 rpart() 中，method = "class" 默认按Gini指数选择分裂特征和切分点。Gini = 1 - Σpk²，值越小表示节点越纯净（单一类别）。与ID3（信息增益）和C4.5（信息增益率）不同，CART使用Gini指数。

(2) minsplit = 3：节点中至少有3个样本才考虑分裂（防止在极小的节点上继续分裂）。cp = 0.01：复杂度参数，分裂必须使模型拟合提升至少0.01才被接受。若数据集有1000个样本但 minsplit=3 太小，树可能过度生长——产生大量叶节点，每个叶节点只包含极少数样本，导致过拟合。通常建议按样本量比例调整 minsplit。

(3) summary(model) 输出中，Node number 部分列出每个分裂节点使用的变量（如 split: 特征1 < 2.25）；Variable importance 列出每个变量对模型的重要性（基于分裂中Gini减少量的总和）。通过这两部分可以清楚识别决策树使用了哪些变量。
