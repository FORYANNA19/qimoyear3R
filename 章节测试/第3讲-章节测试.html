第3讲 章节测试：K近邻法 K近邻法

共5种题型：选择题(5题) + 填空题(3题) + 简答题(2题) + 计算题(1题) + 编程题(1题)

一、选择题（每题4分，共20分）

1. 关于KNN算法的学习方式，以下哪项描述是正确的？
  1. 它是急切学习（Eager Learning）算法，在训练时构建分类模型
  2. 它是懒惰学习（Lazy Learning）算法，训练阶段仅存储数据
  3. 它只能处理分类任务，不能处理回归任务
  4. 它通过梯度下降最小化损失函数来训练参数
答案：B
解析：KNN是典型的懒惰学习（Lazy Learning）算法，训练阶段几乎不做计算，仅存储训练数据。预测时才进行距离计算和表决。It is NOT an eager learner, can handle regression, and does NOT use gradient descent.
2. 在KNN算法中，若将K值设置得非常小（如K=1），模型最可能表现出哪种现象？
  1. 欠拟合（Underfitting）
  2. 过拟合（Overfitting）
  3. 计算速度大幅变慢
  4. 模型自动忽略异常值
答案：B
解析：K越小，模型越复杂，决策边界越曲折，对噪声极其敏感，极易过拟合。K=1时，每一个训练样本本身都能形成自己的"领地"。
3. 使用KNN前通常需要对特征进行标准化/归一化，主要原因是什么？
  1. 将非线性问题转化为线性可分
  2. 加快寻找最近邻的速度
  3. 处理缺失值
  4. 防止数值范围较大的特征在距离计算中占据主导
答案：D
解析：若特征量纲不一致（如身高mm vs 体重kg），距离计算会被大数值特征主导，小数值特征几乎不起作用，导致KNN失效。
4. 暴力搜索下，对N个训练样本（每个D维特征）的一个新样本用KNN分类，时间复杂度约为？
  1. O(1)
  2. O(log N)
  3. O(N × D)
  4. O(N²)
答案：C
解析：暴力搜索需要计算新样本与所有N个训练样本的距离，每个距离计算需要遍历D个特征，故复杂度为 O(N×D)。
5. 面对极端不平衡数据集（A类99%，B类1%），K值较大时会产生什么问题？
  1. 算法抛出异常
  2. 测试样本极易被预测为多数类A，少数类召回率极低
  3. 测试样本被全部预测为少数类B
  4. 模型自动转换为回归模型
答案：B
解析：在大K值下，多数表决的邻居中多数类占据绝对优势，少数类样本几乎不可能被识别。应对策略：加权投票、调整K值、数据重采样等。
6. "维度灾难"对KNN的致命影响是什么？
  1. 高维空间中数据点距离趋于相近，近邻概念失去分辨力
  2. 使得K值选择不再重要
  3. 导致内存溢出但准确率仍很高
  4. 使数据线性可分
答案：A
解析：维度灾难（Curse of Dimensionality）：随着维度p增加，覆盖r比例邻居的超立方体边长 ∝ r^(1/p)趋于1，意味着几乎所有点都落在某点的"邻域"内，距离失去了区分力。

二、填空题（每题4分，共12分）

7. K近邻法的三个基本要素是：K值选择、距离度量 和 分类决策规则。
答案：K值选择、距离度量、分类决策规则
解析：这三个要素共同决定了KNN的预测行为。确定后，对任何新输入，输出唯一确定。
8. 在0-1损失函数下，多数表决规则等价于 经验风险最小化。
答案：经验风险最小化
解析：要使误分类率最低，等价于使邻域内正确分类的样本数最多，即多数表决规则。
9. 偏差-方差分解公式为：Err = Bias² + Variance + σ²。其中，偏差刻画了学习算法的拟合能力，方差刻画了数据扰动造成的影响。
答案：Err = Bias² + Variance + σ² ；偏差；方差
解析：偏差衡量期望预测与真实值的偏离（拟合能力），方差衡量训练集变动导致的学习性能变化（数据扰动影响），σ²为不可约噪声。

三、简答题（每题10分，共20分）

10. 简述K值的选择对KNN模型的影响，以及如何选取合适的K值。
参考答案：
K较小（如K=1）：模型复杂，决策边界曲折，对噪声敏感，训练误差低但泛化误差可能很高 → 过拟合。
K较大（如K=N）：模型简单，决策边界平滑，训练误差高 → 欠拟合（预测恒为多数类）。
选取方法：通常采用K折交叉验证法，对候选K值分别计算交叉验证误差，选择误差最小的K。二分类问题建议K取奇数以避免平票。
11. 什么是"维度灾难"？它如何影响KNN，有哪些应对策略？
参考答案：
定义：随着特征空间维度p的增加，数据点之间的距离趋于相近，"最近邻"的概念失去分辨力。
数学表达：对于原点，覆盖r比例邻居的超立方体的期望边界长度 ∝ r^(1/p)，当p很大时，该边长趋近于1，意味着几乎包含所有数据点。
影响：KNN在高维空间中预测误差显著增大，局部性丧失。
应对策略：①特征选择（保留离散度大、相关性强的特征）；②降维（PCA等）；③前向序列选择。

四、计算题（20分）

12. 已知二维空间中的三个点：x₁=(1,1), x₂=(5,1), x₃=(4,4)。

(1) 计算 x₁ 到 x₂ 和 x₃ 的欧氏距离，确定最近邻。（5分）

(2) 计算 x₁ 到 x₂ 和 x₃ 的曼哈顿距离，确定最近邻。（5分）

(3) 若 p≥3，Lp 距离下 x₁ 的最近邻是哪个？简要说明原因。（5分）

(4) 上述现象说明了什么关于距离度量的结论？（5分）

参考答案：
(1) 欧氏距离 (p=2)：
d₂(x₁,x₂) = √((1-5)²+(1-1)²) = √16 = 4.0
d₂(x₁,x₃) = √((1-4)²+(1-4)²) = √(9+9) = √18 ≈ 4.24
∵ 4.0 < 4.24, ∴ 最近邻为 x₂

(2) 曼哈顿距离 (p=1)：
d₁(x₁,x₂) = |1-5|+|1-1| = 4.0
d₁(x₁,x₃) = |1-4|+|1-4| = 3+3 = 6.0
∵ 4.0 < 6.0, ∴ 最近邻为 x₂

(3) p≥3时，最近邻为 x₃。
原因：L₃(x₁, x₃) ≈ 3.78, L₄(x₁, x₃) ≈ 3.57，随着p增大，Lp(x₁, x₃)递减并趋近于3（切比雪夫距离 max(|1-4|,|1-4|)=3），而Lp(x₁, x₂)恒为4。当Lp(x₁, x₃) < 4时，x₃成为最近邻。

(4) 不同距离度量会产生不同的最近邻。距离度量是KNN的核心要素之一，选择哪种距离取决于具体应用场景和数据特性。没有"最好"的距离，只有"最合适"的距离。

五、代码分析题（12分）

13. 阅读以下R语言代码，回答问题：
library(class) # 标准化训练数据（KNN对量纲敏感） train_scale <- scale(iris[1:100, 1:4]) test_sample <- scale(iris[101, 1:4], center = attr(train_scale, "scaled:center"), scale = attr(train_scale, "scaled:scale")) # KNN分类预测 pred <- knn(train = train_scale, test = test_sample, cl = iris[1:100, 5], k = 5) print(paste("预测类别为:", pred)) # 留一交叉验证 pred_cv <- knn.cv(train = train_scale, cl = iris[1:100, 5], k = 5) err <- mean(pred_cv != iris[1:100, 5]) * 100 print(paste("留一法错判率:", round(err, 2), "%"))

(1) 代码中 scale() 函数的作用是什么？为什么在KNN中要使用它？（4分）

(2) k = 5 的含义是什么？如果改为 k = 1 或 k = 100（等于训练样本数），分别会导致什么问题？（4分）

(3) knn.cv() 函数与 knn() 函数有什么区别？它如何评估模型性能？（4分）

参考答案：
(1) scale() 对数据进行标准化（中心化+缩放），使每个特征的均值为0、标准差为1。KNN基于距离度量，如果特征量纲不一致（如"身高cm"与"体重kg"），距离计算会被大数值特征主导，小数值特征几乎不起作用。标准化消除了量纲影响。

(2) k = 5 表示取最近5个邻居进行投票分类。若 k = 1，模型过于复杂、对噪声敏感，极易过拟合（回代法误差为0%）。若 k = 100（全部样本），模型过于简单，预测恒为多数类，导致欠拟合。

(3) knn.cv() 是class包中专门用于留一交叉验证的函数，自动对每个样本：用除自身外的n-1个样本训练、对自身预测，循环n次后统计总错判率。knn() 则需要显式指定独立的训练集和测试集。knn.cv() 的评估结果更接近真实泛化误差。
