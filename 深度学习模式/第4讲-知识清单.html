1. 决策树模型核心概念 细读

1.1 直觉 (Intuition) 细读

就像我们日常生活中做决策——先问"年龄多大？"再问"是不是学生？"一步步缩小范围，最终得出结论。 决策树以树结构形式进行决策，每个内部节点代表一个属性测试，每个分支代表测试结果，每个叶节点代表分类标签。

1.2 定义 (Definition) 细读

决策树 (Decision Tree)是以树结构（包括二叉树和多叉树）形式来表达的预测分析模型， 常用于解决分类和回归问题。

根节点 (Root node) — 包含样本全集的最顶层节点
内部节点 (Internal node) — 对应某个属性或特征的测试
叶节点 (Leaf node) — 代表分类后所获得的分类标记
分支 (Branch) — 对应于属性的某个取值

1.3 本质理解 细读

决策树本质上是对特征空间的一种完备划分，每一种划分对应一种决策规则。 从根到叶的每条路径构成一条 if-then 规则：内部节点 = 条件，叶节点 = 结论。

重要性质：任意两条路径之间互斥且完备——任意一个实例恰好被一条路径所覆盖。

决策树是判别模型：学习的是条件概率分布 P(Y|X)，而非联合分布 P(X,Y)。 将特征空间划分为互不相交的单元，每个单元给出类的条件概率分布。

2. 决策树构建的四个关键问题 细读

  1. 特征选择 — 如何选择最优划分属性？（信息增益 / 增益率）
  2. 叶节点类别确定 — 最终叶结点对应的分类标签如何确定？
  3. 树构建算法 — 自上而下的递归分裂（ID3 / C4.5）
  4. 剪枝 (Pruning) — 防止过拟合，提高泛化能力

3. 特征选择：熵与信息增益 细读

3.1 熵 (Entropy) 细读

熵衡量随机变量的不确定性。熵越大，不确定性越大。

$$H(D) = -\sum_{k=1}^{K} p_k \log_2 p_k$$
其中 pk = |Ck| / |D|，即第 k 类样本所占比例。底数为 2 时单位为 bit。

极端情况： 当所有样本属于同一类时，H(D) = 0（不确定性最小）；当各类均匀分布时，H(D) 最大 = log2K。

3.2 条件熵 (Conditional Entropy) 细读

表示在已知随机变量 X 的条件下，随机变量 Y 的不确定性。

$$H(D \mid A) = \sum_{i=1}^{n} \frac{|D_i|}{|D|} \cdot H(D_i)$$
按特征 A 取值将 D 划分为 n 个子集 Di，对每个子集加权求和。权重为 |Di|/|D|（分支样本越多，权重越大）。

3.3 信息增益 (Information Gain) 细读

信息增益 = 得知特征 A 的信息后，类别 Y 不确定性减少的程度。又称互信息 (Mutual Information)。

$$g(D, A) = H(D) - H(D \mid A)$$
信息增益 = 划分前熵 − 划分后条件熵

ID3 准则：选择信息增益最大的特征作为当前节点的划分属性。

3.4 信息增益计算步骤 细读

  1. 计算数据集 D 的经验熵 H(D)（划分前的信息熵）
  2. 对每个特征 A：
    • 计算每个取值 A=ai 下的经验条件熵 H(Di)
    • 加权求和得到 H(D|A)（划分后的信息熵）
  3. 计算信息增益：g(D,A) = H(D) − H(D|A)

4. ID3 算法 细读

4.1 基本思想 细读

从根节点开始，自顶向下递归构建。每次以信息增益为准则选择最优划分属性，按其取值建立分支。 目标是构建一棵熵值下降最快的树，希望叶子节点处熵值为 0（纯净）。

4.2 三种停止条件 细读

条件情形处理
(1)当前节点样本全属于同一类别将该节点标记为叶节点，类别为该类
(2)当前属性集为空（用完所有属性）标记为叶节点，类别为多数类
(3)当前节点样本集合为空标记为叶节点，类别为父节点多数类

4.3 算法策略 细读

贪心算法 (Greedy Algorithm)：每一步选择当前最优（信息增益最大）的属性，不回溯。

4.4 决策边界特性 细读

决策树的决策边界总是与坐标轴平行或垂直——形成阶梯状的划分。 这是因为每次分裂仅在单一特征上做条件判断。

5. C4.5 算法与信息增益率 细读

5.1 为什么需要 C4.5？ 细读

ID3 的信息增益偏向选择取值较多的特征。例如将"编号"作为属性——每个样本的编号唯一，条件熵 H(D|编号)=0，信息增益 = H(D) 达到最大，但这样的划分毫无泛化能力，树庞大且极浅，根本不合理。

ID3 偏向取值多（如身份证号、日期）的特征 → C4.5 引入信息增益率进行修正。

5.2 信息增益率 (Gain Ratio) 细读

$$g_R(D, A) = \frac{g(D, A)}{H_A(D)}$$

其中分母 HA(D) 是特征 A 的固有值 (Intrinsic Value)（即用 D 的样本分布来度量 A 本身的熵）：

$$H_A(D) = -\sum_{i=1}^{n} \frac{|D_i|}{|D|} \log_2 \frac{|D_i|}{|D|}$$
特征 A 取值越多 → HA(D) 越大 → 增益率被惩罚。

5.3 C4.5 对缺失值的处理 细读

当某样本在特征 A 上缺失时，C4.5 根据已知该特征权重的比例，将样本分配到所有子节点中（按概率分配），而非直接丢弃或简单填充。

5.4 连续值处理 细读

决策树处理连续型特征（如年龄、收入）使用二分法 (Bi-partition)：将连续值排序后，取相邻值的中间点作为候选划分点，寻找最优划分点。

7. 决策树剪枝 细读

7.1 过拟合问题 细读

决策树递归生长到无法继续为止 → 对训练数据分类很准确，但对未知测试数据不准确（过拟合）。 原因：数据有噪声、训练样本太少 → 为了正确分类所有训练样本，分支过多。

7.2 两种剪枝策略 细读

策略机制优点缺点
预剪枝
(Pre-pruning)		 提前停止分裂：限定最大深度 / 叶节点最小样本量 计算效率高 可能欠拟合（错过后续更好的分裂）
后剪枝
(Post-pruning)		 先生成完整树，再自下而上剪去不具代表性的子树 泛化性能更优 计算开销较大

7.3 后剪枝的损失函数 细读

定义决策树 T 的损失函数为结构化风险：

$$C_\alpha(T) = C(T) + \alpha |T|$$

其中：

  • C(T) = ∑t Nt · Ht(T) —— 模型对训练数据的拟合程度（经验风险）
  • |T| —— 叶节点个数，衡量模型复杂度
  • α ≥ 0 —— 调节参数，平衡拟合与复杂度

α 越小 → 决策树越大（偏向拟合）；α 越大 → 决策树越小（偏向简单）。

剪枝目标：给定 α 后，选择使 Cα(T) 最小的子树。
