1. 模拟实验概述 高频 细读

1.1 实验目的与直觉 细读

核心目标：观察参数 $K$ 取不同值时对K-近邻法分类边界和错判率的影响。 KNN的预测结果高度依赖于 $K$ 的选择：$K$ 太小则模型过于敏感、容易过拟合噪声；$K$ 太大则分类边界过度平滑、可能忽略局部模式。本模拟通过计算 $K\in[1,30]$ 范围内的错判率曲线，直观展示这一权衡关系。

直觉类比：$K$ 就像「听取多少人意见」。$K=1$ 只听最近的一个邻居，容易被个别噪音带偏（过拟合）；$K=60$ 听取所有人，结果趋于大多数（欠拟合）。要找的是那个「刚好合适」的邻里规模。

1.2 数据生成 高频

X1 和 X2 均在 $[-1,+1]$ 范围内均匀分布生成60个随机数。 两个特征相互独立，覆盖二维平面上的一个正方形区域。 Y 按照 0.7:0.3 的比例随机指派类别 1 和 0（偏斜的两类分布）。 这意味着类别1约占70%（42个），类别0约占30%（18个）。 

set.seed(12345)
x1 <- runif(60, -1, 1)
x2 <- runif(60, -1, 1)
y <- sample(c(0, 1), size = 60, replace = TRUE, prob = c(0.3, 0.7))
data_clf <- data.frame(Fx1 = x1, Fx2 = x2, Fy = as.factor(y))

1.3 训练集/测试集划分 细读

从60个样本中随机抽取18个（30%）作为测试集，剩余42个（70%）为训练集。 随机抽样的目的是保证训练集和测试集都来自同一分布，使评估结果具有代表性。 

sample_id <- sample(x = 1:60, size = 18)
data_test <- data_clf[sample_id, ]
data_train <- data_clf[-sample_id, ]
易错：测试集比例通常取 20%~30%，过小则评估方差大、不稳定；过大则训练数据不足、模型学不到位。本实验取30%（18个）是合理的折中。

2. KNN分类预测模拟分析 必考 细读

本章计算 $K\in[1,30]$ 时使用三种不同评估方式得到的错判率曲线，是理解KNN模型评估的核心内容。

2.1 三种评估方法详解 必考

方法中文训练数据测试数据特点
Resubstitution 回代法 全量数据 全量数据 用同一份数据既训练又测试，严重低估真实误差，因为模型已经"见过"测试数据。$K=1$ 时回代误差率为 0%（完美记忆所有样本）。
Holdout 留出法 训练集（42个） 测试集（18个） 训练和测试数据不重叠，评估相对客观。但由于只用了一次划分，误差估计的方差较大，依赖于具体的划分方式。
留一交叉验证 (LOOCV) 留一交叉验证 n-1个样本（循环） 剩余1个样本 每次留一个样本做测试，其余做训练，重复 $n$ 次取平均。评估最稳定、偏差最小，但计算开销大（需训练 $n$ 次）。

错判率公式： $$ \text{Error Rate} = \frac{1}{n} \sum_{i=1}^{n} \mathbb{I}(\hat{y}_i \neq y_i) \times 100\% $$ 其中 $\mathbb{I}(\cdot)$ 是指示函数，当预测类别 $\hat{y}_i$ 与真实类别 $y_i$ 不同时取1，否则取0。

2.2 R代码实现：三种方法错判率计算 重点

核心逻辑：对 $K=1$ 到 $30$ 循环，每种方法各自调用 class::knn 或 knn.cv 进行预测，计算误判比例。

err_resub <- numeric(30)
err_holdout <- numeric(30)
err_loocv <- numeric(30)
set.seed(12345)

for(i in 1:30) {
  # 1. 回代法 (Resubstitution) — 全量数据训练 & 全量数据测试
  fit_resub <- knn(train = data_clf[, 1:2], test = data_clf[, 1:2],
                    cl = data_clf[, 3], k = i)
  err_resub[i] <- mean(fit_resub != data_clf[, 3]) * 100

  # 2. 留出法 (Holdout) — 训练集训练 & 测试集测试
  fit_holdout <- knn(train = data_train[, 1:2], test = data_test[, 1:2],
                      cl = data_train[, 3], k = i)
  err_holdout[i] <- mean(fit_holdout != data_test[, 3]) * 100

  # 3. 留一法 (留一交叉验证 (LOOCV)) — 使用 knn.cv 自动执行 留一交叉验证 (LOOCV)
  fit_loocv <- knn.cv(train = data_clf[, 1:2], cl = data_clf[, 3], k = i)
  err_loocv[i] <- mean(fit_loocv != data_clf[, 3]) * 100
}
易错：knn() 的 train 和 test 参数只接受数值矩阵/数据框，不包含类别列；类别列通过 cl 参数单独传入。回代法中 train 和 test 指向同一数据，这是故意的——就是在同一份数据上评估。

2.3 结果可视化 重点

将三条错判率曲线画在同一张图上，横轴为 $K$，纵轴为错误率（%），便于对比三种方法的评估行为差异。 

error_data <- data.frame(
  K = 1:30,
  Resubstitution = err_resub,
  Holdout = err_holdout,
  留一交叉验证 (LOOCV) = err_loocv
) %>% pivot_longer(cols = -K, names_to = "Method", values_to = "ErrorRate")

ggplot(error_data, aes(x = K, y = ErrorRate, color = Method, linetype = Method)) +
  geom_line(size = 1) + theme_minimal() +
  labs(title = "Error Rate vs. K Value by Evaluation Method",
       y = "Error Rate (%)", x = "K") +
  scale_y_continuous(limits = c(0, 80))

2.4 K值对错判率的影响规律 必考

三条曲线的典型表现：

  • 回代法 (Resubstitution)：$K=1$ 时误判率为 0%，随着 $K$ 增大单调上升。因为 $K=1$ 时每个样本的最近邻就是自己，完美分类自身；$K$ 越大，决策边界越平滑，越难以完美拟合训练数据。
  • 留出法 (Holdout)：曲线呈 U 形或波动下降后回升。$K$ 很小时方差大（对测试集中的个别点敏感），$K$ 增大到一定程度后误差趋于稳定或轻微上升。曲线波动较大，因为测试集只有18个样本，估�误差的方�本身较高。
  • 留一法 (留一交叉验证 (LOOCV))：在 $K$ 较小时误差较高（过拟合），随着 $K$ 增加先下降后趋于平坦或缓慢上升。曲线最平滑、最可靠，因为每个样本都被用作测试一次，评估最全面。
易错：三条曲线的 绝对数值不可直接比较——它们评估的是不同训练/测试配置下的表现。回代法必然给出最乐观的估计（最低误差），LOOCV给出最无偏的估计。考试常问：哪种方法的误差估计最接近真实泛化误差？答案是 留一交叉验证 (LOOCV)。

K值选择的核心权衡：

  • $K$ 太小（如 $K=1\sim3$）：模型方差大，决策边界凹凸不平，容易过拟合训练数据中的随机噪声。在LOOCV和Holdout曲线上表现为高错误率。
  • $K$ 太大（如 $K\rightarrow n$）：模型偏差大，决策边界过于平滑，可能忽略真实的局部模式。回代法误差逼近先验概率分布所决定的基础误差率。
  • 最优 $K$：通常通过交叉验证（如LOOCV）选择使错误率最低的 $K$ 值。本模拟中，LOOCV的优值通常在 $K=5\sim15$ 范围内。

3. 三种评估方法对比 必考 细读

维度回代法 (Resubstitution)留出法 (Holdout)留一法 (留一交叉验证 (LOOCV))
偏差 严重低估（高偏差） 中等（受划分影响） 几乎无偏
方差 低（数据量最大） 高（依赖单次划分） 较高（样本少时）
计算开销 $O(1)$ 次训练 $O(1)$ 次训练 $O(n)$ 次训练
适用场景 仅作下界参考 数据量大、快速评估 小样本、需精确评估

易错：不要把「回代法误差率低」当作模型好——恰恰相反，回代法误差率越低说明模型过拟合越严重（尤其是 $K=1$ 时回代误差为0）。考试常设陷阱题：给出一张图让学生判断哪条曲线对应哪种方法，关键看 $K=1$ 时哪条在0%附近。

4. KNN回归预测模拟分析 重点 细读

KNN不仅可以做分类，还可以做回归。回归的目标变量 $Y$ 是连续数值型。与分类的关键区别在于：分类用多数投票决定预测类别，回归用 $K$ 个最近邻的目标值取平均作为预测值。

4.1 回归数据生成 细读

  • X1：$[-1,+1]$ 均匀分布
  • X2：标准正态分布 $N(0,1)$
  • Y：$[10,20]$ 均匀分布连续值

4.2 回归预测公式 细读

KNN回归的预测值等于 $K$ 个最近邻目标值的算术平均： $$ \hat{y} = \frac{1}{K} \sum_{i=1}^{K} y_{(i)} $$ 其中 $y_{(i)}$ 是按距离排序后第 $i$ 近的邻居的目标值。也可以使用距离加权平均（如核函数加权），但基础版本就是简单平均。

4.3 R代码（caret::knnreg）高频

注意：分类任务使用 class::knn，回归任务必须使用 caret::knnreg（或 FNN::knn.reg），否则 class::knn 会将连续目标变量强制转为无意义的因子。

library(caret)
mse_vector <- numeric(30)
for(i in 1:30) {
  knn_reg_model <- knnreg(Fy ~ Fx1 + Fx2, data = data_train_reg, k = i)
  preds <- predict(knn_reg_model, newdata = data_test_reg)
  mse_vector[i] <- mean((data_test_reg$Fy - preds)^2)
}

4.4 回归评估指标：均方误差 (MSE) 重点

均方误差（Mean Squared Error）： $$ \text{均方误差 (MSE)} = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2 $$ 均方误差 (MSE) 衡量预测值与真实值之间的平均平方距离。均方误差 (MSE) 越小越好，对异常值敏感（因为平方放大了大误差的影响）。在KNN回归中，均方误差 (MSE) vs K 的曲线通常也呈 U 形，最优 $K$ 取 均方误差 (MSE) 最低点。

易错：分类用「错判率」（Error Rate），回归用「均方误差 (MSE)」或 RMSE。两者不可混淆——分类任务的评估指标是0-1损失的比例，回归任务是连续误差的平方均值。考试中若把回归的MSE公式套到分类题上，拿不到分。

5. 案例分析：预测顾客消费决策 重点 细读

实战KNN建模全流程：数据预处理 → 超参数调优 → 模型评估 → 变量重要性分析。

5.1 数据预处理 高频

KNN计算距离时，数值型变量必须先标准化（center + scale），否则量纲大的变量会主导距离计算。 例如 Age 的范围是 20~60，Income 的范围是 20000~80000——不标准化的�况下，Income 对距离的贡献远大于 Age，导致模型被 Income 主导。 

real_data$Income <- as.factor(real_data$Income)
real_data$Gender <- as.factor(real_data$Gender)
real_data$Purchase <- as.factor(real_data$Purchase)

5.2 使用 caret + kknn 进行超参数调优 必考

caret 框架提供统一的训练接口，kknn 包支持带核函数加权的KNN。 使用 10 折交叉验证（10-fold CV）自动搜索最优的超参数组合。 

train_control <- trainControl(method = "cv", number = 10)
set.seed(123)

knn_grid <- expand.grid(
  kmax = seq(3, 25, by = 2),       # 最大邻居数
  distance = 2,                     # 闵可夫斯基距离阶数 (2=欧氏距离)
  kernel = c("rectangular", "triangular", "gaussian")
)

knn_model <- train(
  Purchase ~ Age + Gender + Income,
  data = real_train,
  method = "kknn",
  trControl = train_control,
  tuneGrid = knn_grid,
  preProcess = c("center", "scale")  # 标准化
)

5.3 调优结果解读 细读

最佳参数组合由 knn_model$bestTune 输出。 本例中最佳配置为 kmax=5, kernel=triangular, distance=2。交叉验证准确率曲线随 kmax 增加先升后降，triangular 核通常表现优于 rectangular（不加权）和 gaussian。

5.4 变量重要性 了解

varImp() 函数基于 ROC 曲线下面积评估各变量对分类的贡献。 在本案例中，Income 是最重要的预测变量（重要性约100），其次是 Gender，最后是 Age。这说明顾客的收入水平对消费决策的影响最大。 

importance <- varImp(knn_model, scale = TRUE)
plot(importance, main = "kNN Variable Importance (ROC curve area)")
易错：变量重要性在KNN中不如在树模型中可靠。KNN是"懒学习"方法，没有显式的参数权重，varImp 是通过扰动变量的方式间接评估的，结果仅供参考，不可过度解读。
