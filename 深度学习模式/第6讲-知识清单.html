第6讲：决策树 — Bagging与随机森林

一、Bagging 集成学习 必考 细读

1.1 直觉理解 (Intuition) 细读

想象一个班级投票决定去哪儿春游。如果只问一个人，可能偏颇；但如果从班级中随机抽样多次组成多个小组，每组独立投票，最后统计所有小组的投票结果 —— 这就是 Bagging 的思想。通过"群体的智慧"（Wisdom of Crowds）来降低个体决策的随机误差。

1.2 核心定义 (Definition) 细读

Bagging（Bootstrap Aggregating）是一种并行式集成学习算法，通过对原始训练集进行Bootstrap抽样（有放回抽样）生成多个不同的训练子集，在每个子集上独立训练一个基学习器，最后通过投票（分类）或平均（回归）的方式组合预测结果。

Bootstrap Aggregating = Bootstrap（自助抽样）+ Aggregating（聚合），名字本身就说明了它的两个核心步骤。

1.3 动机与来源 (Motivation) 细读

Bagging 由 Leo Breiman 于 1996 年提出。核心动机：

  • 降低方差：单个决策树对训练数据的微小变化很敏感（高方差），Bagging 通过对多个树的预测取平均来平滑这种波动。
  • 利用 Bootstrap 的多样性：有放回抽样使每棵树看到的训练集都不同（约63.2%原始样本），从而产生多样化的基学习器。
Bagging 主要降低方差而非偏差。如果基学习器本身偏差很大（如欠拟合），Bagging 不会显著改善性能。因此 Bagging 通常搭配"强而高方差"的基学习器（如未剪枝的决策树）。

1.4 算法推导 (Derivation / Steps) 细读

输入：训练集 D 含 N 个样本；基学习算法（如决策树）；迭代次数 B

输出：集成模型 H(x)

步骤操作说明
Step 1Bootstrap 抽样从 D 中有放回地抽取 N 个样本，形成训练子集 Db
Step 2训练基学习器在 Db 上训练一个基学习器 hb(x)（如决策树）
Step 3重复 B 次重复 Step 1~2，得到 B 个基学习器 {h1, h2, ..., hB}
Step 4聚合预测分类：多数投票；回归：简单平均

1.5 Bootstrap 抽样细节 细读

从 N 个样本中有放回抽样 N 次：

每个样本被选中的概率：1/N
每个样本未被选中的概率：(1 - 1/N)N
当 N → ∞ 时：(1 - 1/N)N ≈ 1/e ≈ 0.368
∴ 每次 Bootstrap 约有 63.2% 的样本被选中（袋内样本），36.8% 未被选中（袋外样本 袋外 (OOB)）

1.6 Bagging 的数学表达 细读

分类： H(x) = argmaxy Σb=1B I(hb(x) = y)
回归： H(x) = (1/B) Σb=1B hb(x)

二、随机森林 (Random Forest) 必考 细读

2.1 直觉理解 (Intuition) 细读

Bagging 让每棵树看到不同的样本；随机森林进一步让每棵树的每个节点只能看到特征的随机子集来分裂。双重随机性使树与树之间的相关性大大降低，从而进一步降低集成模型的方差。可以理解为：Bagging 是"让每棵树读不同的书"，随机森林是"让每棵树不仅读不同的书，还只能看不同章节的摘要"。

2.2 核心定义 (Definition) 细读

随机森林（Random Forest）是在 Bagging 基础上增加特征随机选择的集成学习算法。在每棵决策树的每个节点分裂时，不是从全部 M 个特征中选最优，而是先随机抽取 m 个特征（m ≪ M），再从这 m 个特征中选最优分裂特征。

"Forest" = 多棵树 = Bagging；"Random" = 随机样本（Bootstrap）+ 随机特征（节点分裂时的特征子集）。两个随机性缺一不可！

2.3 算法步骤 (Algorithm) 细读

步骤内容
Step 1对 b = 1, 2, ..., B（共 B 棵树），从 N 个样本中有放回地随机抽取 N 个样本作为第 b 棵树的训练集
Step 2在每个节点分裂时：① 从全部 M 个特征中随机选择 m 个特征（m ≪ M）；② 从这 m 个特征中按某种策略（如基尼指数/信息增益）选择最优分裂特征和切分点；③ 分裂为两个子节点
Step 3递归执行 Step 2 直到满足停止条件，生成一棵完整的决策树（通常不剪枝）
Step 4重复 Step 1~3 共 B 次，构建 B 棵决策树，形成随机森林
Step 5预测：分类 → 多数投票；回归 → 取平均

2.4 关键超参数 细读

参数含义典型取值影响
B (ntree)树的数量通常 500~1000越多越稳定，但边际收益递减
m (mtry)每次分裂随机选取的特征数分类：√M；回归：M/3m 越小，树间相关性越低，但单树强度也越低
每棵树的最大深度限制树的生长通常不限制（充分生长）树太浅会欠拟合
当 m = M（即不随机选特征），随机森林退化为普通的 Bagging。随机森林的核心优势正是来自 m ≪ M 带来的特征随机性，使各树之间"去相关"（decorrelation）。

2.5 随机森林 vs Bagging 对比 细读

维度BaggingRandom Forest
随机性来源仅 Bootstrap 样本（行抽样）Bootstrap 样本 + 特征子集（行+列抽样）
树间相关性较高更低（特征随机减少了相关性）
方差较低（相对单树）更低
基学习器强度较强略弱（因特征受限），但集成后整体更强
计算效率较慢更快（因每节点只搜索 m 个特征）

三、偏差-方差分解 (Bias-Variance Decomposition) 重点 细读

3.1 核心公式 细读

对于回归问题，一个模型的泛化误差可以分解为三部分：

E[(y - f̂(x))²] = (Bias[f̂(x)])² + Var[f̂(x)] + σ²

其中：
Bias[f̂(x)] = E[f̂(x)] - f(x)  （偏差：模型期望预测与真实值的差距）
Var[f̂(x)] = E[(f̂(x) - E[f̂(x)])²]  （方差：模型预测的波动程度）
σ²  （不可约误差：数据本身的噪声）

3.2 三种误差的含义 细读

误差类型含义反映什么降低方式
偏差 (Bias)模型预测值的期望与真实值之间的差距模型本身的拟合能力（准确度）增加模型复杂度（如更深的树）
方差 (Variance)用不同训练集训练出的模型，预测值之间的差异模型对数据波动的敏感性（稳定性）集成学习（Bagging/RF）、正则化、增加数据
噪声 (Noise)数据本身的随机波动问题本身的难度下限无法通过优化模型降低

3.3 偏差-方差权衡 (Bias-Variance Tradeoff) 细读

模型偏差方差说明
单棵决策树（未剪枝）低高容易过拟合
Bagging（决策树基学习器）≈ 单棵决策树显著降低平均操作降低了方差，偏差几乎不变
随机森林略增（相对 Bagging）进一步降低特征随机使树间去相关，方差更低
AdaBoost / Boosting显著降低可能略有增加Boosting 主要降低偏差（见后续章节）
Bagging 和随机森林主要降低方差（适合高方差的基学习器）；而 Boosting（如 AdaBoost）主要降低偏差（将弱学习器逐步提升为强学习器）。这是两者最本质的区别。

3.4 数学直觉 细读

假设 B 棵树，每棵方差为 σ²，两两相关系数为 ρ，则集成模型的方差为：

Var(H) = ρ·σ² + (1-ρ)·σ²/B
当 ρ → 0：Var(H) ≈ σ²/B（理想去相关，方差除以 B）
当 ρ → 1：Var(H) ≈ σ²（所有树一样，集成无增益）

随机森林通过特征随机性降低树间相关性 ρ，从而降低整体方差。

四、袋外误差 (袋外误差 (OOB Error)) 重点 细读

4.1 定义 细读

袋外误差（Out-of-Bag Error, 袋外误差 (OOB Error)）是利用 Bootstrap 抽样中未被选中的样本（约36.8%）作为验证集来估计模型泛化误差的方法。无需额外划分验证集，是一种"免费"的交叉验证。

4.2 计算方式 细读

类型计算公式
袋外 (OOB) 分类误差对每个样本 i，用未包含该样本的树（袋外树）投票，计算误分类比例
袋外 (OOB) 回归误差对每个样本 i，用 OOB 树预测值的平均与真实值的均方误差

4.3 OOB 误差的用途 细读

  • 评估模型泛化性能，无需额外验证集
  • 用于调参（如选择最优的 m 和 B）
  • 用于特征重要性度量（通过置换 OOB 样本的特征值，观察 OOB 误差的变化）
OOB 误差不是测试集误差，它是"内嵌的"交叉验证误差，可以提供无偏估计，但不能完全替代独立测试集的评估。

五、AdaBoost 必考 扩展 细读

5.1 核心思想 细读

AdaBoost（Adaptive Boosting）是一种串行式集成学习算法。核心思想：

  1. 每一轮根据上一轮的分类结果，提高被错分样本的权重，降低被正确分类样本的权重
  2. 通过加权多数表决组合弱分类器：误差率小的分类器获得更大权重
与 Bagging 的"并行独立训练"不同，AdaBoost 的分类器之间有依赖关系：后一个分类器的训练依赖于前一个分类器的结果。

5.2 理论基石 细读

核心定理（Schapire, 1989）：在PAC学习框架下，强可学习 ⇔ 弱可学习。即只要存在一个比随机猜测好的弱学习算法，就可以将其提升为强学习算法。这是所有 Boosting 方法的理论基础。

5.3 算法步骤 细读

步骤内容公式
初始化对 N 个样本赋予均匀权重wi(1) = 1/N, i = 1,2,...,N
(a)使用当前权重分布训练弱分类器 Gm(x)—
(b)计算分类误差率em = Σ wi(m) · I(Gm(xi) ≠ yi)
(c)计算分类器权重系数αm = ½ ln((1-em)/em)
(d)更新样本权重（错分增加，正确减少）wi(m+1) ∝ wi(m) · exp(-αm · yi · Gm(xi))
最终分类器加权投票G(x) = sign(Σ αm · Gm(x))

5.4 αm 的性质 细读

emαm含义
em = 0（完美分类）αm → +∞分类器被赋予极大权重
em < 0.5（比随机好）αm > 0分类器有正面贡献
em = 0.5（等于随机）αm = 0分类器无贡献，权重为0
em > 0.5（比随机差）αm < 0反向使用该分类器即可

5.5 AdaBoost 的数学本质 细读

AdaBoost 可视为加法模型（Additive Model）+ 指数损失函数 + 前向分步算法的特例：

模型：f(x) = Σ αm · Gm(x)（加法模型）
损失函数：L(y, f(x)) = exp(-y · f(x))（指数损失）
优化方法：前向分步算法（每一步只优化当前轮参数）

5.6 训练误差界 细读

AdaBoost 的训练误差以指数速度下降：

训练误差 ≤ exp(-2 Σ γm²)
其中 γm = ½ - em，表示分类器比随机猜测好的程度。
若对所有 m 有 γm ≥ γ > 0，则训练误差 ≤ exp(-2Mγ²)

六、提升树 (Boosting Tree) 常考 细读

6.1 定义 细读

提升树（Boosting Tree）是以决策树为基函数的提升方法，采用加法模型 + 前向分步算法。被认为是统计学习中性能最好的方法之一。

6.2 分类 vs 回归 细读

类型基函数损失函数与 AdaBoost 关系
分类提升树二叉分类树指数损失即 AdaBoost 的特例（基分类器为二叉分类树）
回归提升树二叉回归树平方误差损失每步拟合残差（residual），即当前模型预测值与真实值的差

6.3 回归提升树的核心：残差拟合 细读

第 m 步需要拟合的目标是：

rmi = yi - fm-1(xi)  （当前模型的残差）
第 m 棵回归树 Tm(x) 对残差进行拟合
fm(x) = fm-1(x) + Tm(x)
回归提升树的每一棵新树都在"修正"前面所有树的累积误差，通过逐步拟合残差来逼近目标函数。
